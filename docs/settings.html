üöÄ Quick Start Guide

Get Snorkel Alert v2 running in under 10 minutes.

1 Fork the Repository

  1. Go to github.com/wpezza/snorkel-alert-v2
  2. Click Fork (top right)
  3. This creates your own copy

2 Add Required Secrets

  1. Go to your repo ‚Üí Settings ‚Üí Secrets and variables ‚Üí Actions
  2. Click New repository secret
  3. Add ANTHROPIC_API_KEY (get from console.anthropic.com)
  4. Add at least ONE notification channel (see tabs below)

3 Enable GitHub Pages

  1. Go to Settings ‚Üí Pages
  2. Source: Deploy from a branch
  3. Branch: gh-pages / root
  4. Your dashboard will be at: https://YOUR-USERNAME.github.io/snorkel-alert-v2/

4 Test It!

  1. Go to Actions tab
  2. Click üåä Beach Forecast
  3. Click Run workflow
  4. Wait ~60 seconds for magic ‚ú®

‚ùì FAQ

How do I add a friend?

Pushover: They buy the app, create account, give you their User Key. Update PUSHOVER_USER_KEY to your_key,their_key

Telegram: Add them to your group chat that has the bot.

Email: Add their email to EMAIL_RECIPIENTS

How much does this cost to run?

‚Ä¢ GitHub Actions: Free (runs ~90 seconds per forecast)
‚Ä¢ Claude API: ~$0.01 per forecast (~$0.15/month with 3x weekly)
‚Ä¢ Pushover: $5 one-time per user
‚Ä¢ Telegram/Email: Free

Can I add more beaches?

Yes! Edit the PERTH_BEACHES list in snorkel_alert.py. Each beach needs name, lat/lon, type (snorkel/beach/both), and shelter directions.

The workflow failed - what do I do?

1. Go to Actions ‚Üí click the failed run ‚Üí check logs
2. Common issues: missing API key, invalid Pushover token
3. Make sure all required secrets are set correctly
